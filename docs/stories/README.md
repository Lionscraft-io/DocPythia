# Feature Stories

This directory contains user stories and feature requests.

## Structure

Each story should include:
- **Context**: Why this feature is needed
- **Problem Statement**: What problem does it solve
- **Acceptance Criteria**: How we know it's done
- **Related Spec**: Link to technical specification in `/docs/specs/`

## Naming Convention

`YYYY-MM-DD-story-name.md`

Example: `2025-10-17-user-documentation-search.md`
